void ldms_watch_final (ldms_watch_p watch)
 Close up all open watch files from this watcher and leave a signal in the file system that the application run is done. Any further calls on watch have undefined results. If this function is not called (e.g. due to crash) the file system will still have enough information for cleanup to proceed by other means safely. Normally, a resource manager will also wipe the /dev/shm file system after a job is done. This call also frees all progress points and any other internal data structures supporting the log, progress, and tables files. Using these objects after this call yields undefined behavior.
